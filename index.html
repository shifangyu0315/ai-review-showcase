Brief · 一句话说清

在网易云音乐播放页，为「A 级声临其境」项目做一个「一键权益使用」浮层——把音效、播放器样式、音质三项本来分散配置的权益打包成一个套装，用户在浮层内一键解锁、整体生效，无需逐项开启。

Design Highlights · 设计特色
  • 一键 × 按需并存 · 默认全部选中一键解锁，权益卡右上角可逐项取消，按钮文案随选择数量动态调整
  • 权益内容即主视觉 · 权益卡本身承担主视觉，主卡居中抬高、其余按数量层叠退后（本 demo 示例：AI 情绪调音师 / 情绪模式 / 超清母带）
  • 760ms 电影感入场 · 权益卡先聚合再展开，无放大 · 无越位回落 · 无末段停顿，末段自然减速
  • 环境色随主卡联动 · 环境色随主卡切换，700–800ms 同步过渡（本 demo 示例：紫红 / 玫红 / 黑金）
  • 按钮三态纯文案 · 一键解锁 → 正在解锁 → 前往体验，居中干净，无箭头 / 无加载点
1.2 竞稿双方案对比
本次准备了 2 个思路完全不同的方案 —— 方案 1 全新套装组合、方案 2 套用线上装扮套装规范。最终选定方案 1，理由见下方产品判断。
方案 1 · 全新套装组合
跳出线上装扮套装规范，为「一键权益使用」浮层全新设计视觉：深色玻璃 + 紫红氛围 + 权益卡叠卡轮播（支持 2 / 3 / 4 项自适应）。主卡明显抬高、其余按数量层叠退后，一次性传达「独立权益 + 整体套装」两层信息。
方案 2 · 套用线上装扮套装规范
直接复用线上装扮套装 SKU 的横向卡片规范和交互，浮层适配。开发成本最低、心智延续，但视觉与售卖入口高度相似，缺少心智区隔和视觉冲击力。
选定方案 1 的核心判断

本期是「一键权益使用」新心智，浮层视觉必须与线上装扮套装规范明确区隔，否则用户会误判为常规售卖入口，稀释「一键沉浸」这个动作的价值感。方案 1 用独立权益卡承载单项信息、用叠卡组合承载套装感，一次性同时透传「三个独立权益」+「一个整体套装」两层信息，是本次评审的最优解。

1.3 需求背景 · 为什么做
当前的问题 · 我们的价值主张 · 本次竞稿边界
Problem · 当前问题
问题 01
权益「散养」，缺套装组合能力
音质、音效、装扮三大权益模块相互独立配置，运营和策划无法把有内在关联的权益（如「沉浸听音套装」）作为一个整体向用户推荐。用户只看到零散单品，感受不到打包体验。
问题 02
使用路径长，无法一键触达
想开启一个完整的沉浸听音体验，用户需要跳转 3 个不同页面、6–9 步操作才能全部开启。每一步都是流失点，权益整体使用率长期低迷。
Value · 带来什么
🙋
用户价值
一次动作解锁完整听感，降低试用门槛，让用户直观感受到套装体验
📈
业务价值
集中曝光会员权益，提升套装整体使用率与感知价值
🎯
产品价值
建立「一键使用」心智，为未来跨品类权益打包铺路

02 · Methodology

AI 应用 · 产品×AI 协作范式

评审真正想看的：怎么用 AI 的？以及，产品在里面到底做了什么？

我把「视觉落地 / 前端实现 / 自动化测试」交给 AI，把「产品判断力 / 审美方向 / 心智取舍」握在自己手里。整个竞稿的节奏是 需求 → AI 生稿 → 产品评审 → AI 收敛 的 8 轮闭环——从暗色沉浸原型一路推到真实场景可交互稿，2 天完成。

四阶段各司其职：产品拍板方向，AI 高保真落地

标题区下移 6px（margin-top: -2px → 4px），不改字号不动卡片；加自动化测试锁定该间距。

V5
AI 收敛

AI 收敛
采用「A 方案·忠实参考版」：中央卡 174×224px 抬高 14px、侧卡 156×202px 露出 38–40%，无旋转；重构 8 项间距参数；删除固定 54% 高度，改内容驱动（实测 52%）。
V6
产品反馈
卡片布局不够贴参考图（中央主卡 + 两侧平行退后）；上下间距要对齐云音乐运营浮层规范。
产品反馈
用真实黑胶播放页截图作为底图，原图不能拉伸、压扁、裁切。
V7
AI 收敛
原图 object-fit: contain 独立层完整覆盖手机舞台；所有模拟控件 display: none；只叠加独立暗色遮罩；新增无变形契约测试。
AI 收敛
单素材双层自适应画框：同一 URL 同时生成清晰前景（contain 居中）+ 模糊放大背景（cover 自动取色），不加白底、不二次抠图；未来更换素材复用同规则。
V8
产品反馈
播放器样式素材由服务端返回（178×306 RGBA PNG），运营不额外配置裁切图 / 背景图 / 主题色。
2.3 后续能力沉淀
评审版交付后，产品 × AI 又协作打磨的三项能力优化 —— 让「一键」心智更完整
01
支持按需选择权益 —— 一键 × 按需
从「一键整体使用」走向「一键 + 按需选择」，让用户在浮层内就能自由取舍。
  • 三张权益卡右上角增加小尺寸勾选控件
  • 默认三项权益全部选中
  • 支持独立取消或重新选择，至少保留一项权益
  • 按选择数量动态调整按钮文案
  • 底部说明优化为「可按需选择 1–3 项权益 · 解锁后可分别调整」并微调字号
02
新增权益套装入场动画 —— 760ms 电影感
浮层打开时先聚合再展开，让「套装」在开场就完成心智传达。
  • 三张权益卡先以套装聚合状态出现，再展开为当前轮播布局
  • 每次重新打开浮层都会重播
  • 动画期间暂时锁定滑动与勾选，结束后恢复交互并启动自动轮播
  • 最终优化为 760ms 连续展开：无放大 · 无越位回落 · 无末段停顿
  • 无旋转、无素材变形，三张卡片一次到位、末段自然减速
03
支持 2–4 类权益卡自适应 —— 配几项展几张
运营在播放页策略里组合权益（音效 / 播放器样式 / 音质 / 红心动效），浮层根据数量自动切换布局。
  • 2 张 · 主卡居中 + 右侧预览卡，2 个分页点；每 2s 自动互换，手动操作后暂停 4s 再恢复
  • 3 张 · 保持默认叠卡（中央主卡 + 两侧对称露出），交互延续现有版本
  • 4 张 · 主卡 + 左右侧卡 + 右后方外层卡（露出 20–30px），外层卡亮度 / 透明度最低，不抢主卡
  • 入场动画 · 760ms 保持一致：双卡由聚合展开成「主 + 侧」，四卡展开成「四层套装」
  • 勾选逻辑 · 4 张卡在轮播到主卡或露出状态时都可勾选，「至少保留一项」规则不变

03 · Specification

播放页一键权益使用浮层设计规格

设计规格核心章节 · 已完整落到可交互 HTML 原型

3.1 目标

在网易云音乐播放页展示一个严格遵循浮层规范的底部浮层，用高级、简约、有吸引力的视觉呈现一组可一键使用的权益。

本轮 3 项权益：音效 · AI 情绪调音师 / 播放器样式 · 情绪模式 / 音质 · 超清母带。点击主按钮后作为套装整体使用，一级浮层不逐项勾选；竞稿以视觉效果为第一优先级。

3.2 设计依据
  • Figma《浮层》规范 · 主端画布 402×874，保留遮罩、圆角、滑杆
  • POPO 策划稿 · 支持音质 / 音效 / 装扮组合，至少 2 项，一级只做整体使用
  • 4 张视觉参考图 · 独立权益卡 + 中央主卡 + 两侧露出
  • 真实权益截图 · 紫红情绪能量 / 纵向播放器容器 / 黑金频谱
  • 真实黑胶播放页截图 · 作为最终场景底图
3.3 视觉方向
  • 整体 · 深色玻璃浮层 + 紫红主氛围 + 蓝 / 金侧卡 + 香槟金按钮
  • 优先级 · AI 情绪调音师主卡 → 三卡组合轮廓 → 主标题 → 一键解锁按钮 → 辅助文案
  • 环境色随主卡切换 · 紫红 / 玫红 / 黑金，700–800ms 同步过渡
  • 构图 · 中央主卡明显抬高、两侧近乎平行退后，无明显旋转；禁夸张翻转和高强度粒子
3.4 信息架构

浮层从上到下：

  • 顶部滑杆
  • 主标题：一键沉浸此刻
  • 副标题：音效 · 播放器样式 · 超清母带
  • 三张权益卡叠卡轮播区
  • 三个轮播指示点
  • 固定主按钮：一键解锁三项权益

按钮三态均只显示居中文案：一键解锁三项权益 → 正在解锁… → 前往播放页体验，不附加箭头 / 加载点。

3.5 权益卡组件

三张卡统一外框、圆角、内边距、文字层级和光影，内部素材按权益类型选比例：

  • AI 情绪调音师 · 默认主卡，正方形核心概念图 + 「听懂歌曲情绪，定制每一首的听感」
  • 情绪模式 · 服务端返回纵向素材，双层自适应画框（清晰前景 contain + 模糊背景 cover 自动取色）
  • 超清母带 · 正方形黑金频谱概念图 + 「还原录音室级细节与动态」
3.6 卡片布局与轮播
  • 默认顺序：情绪模式 · AI 情绪调音师（中央） · 超清母带
  • 中央主卡 174×224px，侧卡 156×202px，各露出 38–40%，抬高 14px
  • 自动轮播 2 秒切换，手动操作后暂停 4 秒再恢复；浮层关闭 / 页面隐藏 / reduce-motion 时停止
  • 数量自适应布局：2 张 主 + 右侧预览卡（2 个指示点）· 3 张 默认叠卡（现版本）· 4 张 主 + 左右侧卡 + 右后方外层卡（露出 20–30px）
3.7 交互与状态流

默认演示流

浮层从底部进入 → 默认 AI 情绪调音师 主卡 → 自动轮播 2s / 用户手动切换 → 点击 一键解锁三项权益 → 正在解锁… 加载态 → 前往播放页体验 成功态 → 再次点击关闭浮层，回到已解锁的播放页。

竞稿控制

HTML 提供不干扰主画面的演示控制，可切换 2 / 3 / 4 项权益和重置浮层。
