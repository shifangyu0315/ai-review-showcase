# 视频首屏封面设计

## 目标

解决页面首次进入时视频区域显示黑屏的问题，让用户在点击播放前即可看到有效画面。

## 方案

- 从当前 `demo-final.mov` 开头几秒中选取最早出现的、非黑屏且画面稳定的一帧。
- 将该帧导出为独立资源 `video-poster.jpg`。
- 将 `index.html` 中视频标签的 `poster` 属性改为 `./video-poster.jpg`。
- 保留现有视频控件、尺寸、加载策略和手动播放方式，不启用自动播放。

## 页面行为

1. 页面加载后立即显示 `video-poster.jpg`。
2. 用户点击播放后，封面由视频画面取代并正常播放。
3. 视频无法加载时，封面仍作为静态预览保留。

## 验证标准

- 封面来自视频开始后的第一个稳定画面，不是黑帧。
- 页面初次进入时无需点击即可看到封面。
- 点击播放后视频、声音和原生控制条工作正常。
- HTML 引用的封面文件可在线访问。
- 不新增未被页面引用的孤儿素材，也不提交无关文件变化。
